考虑Material for MkDocs

完成了第一次尝试，感觉十分不熟悉如何push，我应该把mkdocs目录下创建的文件放到clone的文件夹里吗？我什么都不知道。

[[2025-02-15]]

先clone到某个目录，然后在那个目录里mkdocs new . 然后编辑完后mkdocs build，再然后mkdocs gh-deploy –force，需要注意的是，要“ensure that the publishing source branch for your GitHub Page is set to gh-pages ”

现在访问1ji0o.github.io就可以看到mkdocs的界面了。

参照Publishing your site - Material for MkDocs

我是只有gh-pages一个branch，每次改完内容需要手动mkdocs gh-deploy，如果是使用github action的话有所不同。

此外，需要安装git，但是它默认不连代理。连代理参见解决 Github port 443 : Timed out - 知乎

git config --global http.proxy http://127.0.0.1:7890 git config –global https.proxy http://127.0.0.1:7890
